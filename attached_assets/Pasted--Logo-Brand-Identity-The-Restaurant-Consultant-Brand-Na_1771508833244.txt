## Logo Design Requirements

### Keep the existing chef hat icon
- The app already has a cyan chef hat outline icon in the nav bar — **keep this exact icon**
- Do NOT redesign or replace the chef hat — it's already associated with the brand
- The chef hat should be placed inside a thin circular border (emblem style)

### Logo Structure
The logo has three elements:
1. **Chef hat icon** inside a circle (existing icon, cyan stroke, no fill)
2. **"The Restaurant"** — serif font (Playfair Display), larger, primary text color (white on dark, dark on light)
3. **"Consultant"** — sans-serif font (DM Sans), small caps, wide letter-spacing (3-4px), uppercase, cyan color

Layout:
```
[Chef hat in circle]  The Restaurant
                      CONSULTANT
```

The icon sits to the left. Text stacks vertically to the right of the icon. "The Restaurant" is the dominant line. "CONSULTANT" sits directly below in smaller, spaced-out uppercase.

### Create Multiple Sizes

**Full logo (landing page, marketing):**
- Icon: 52px circle
- "The Restaurant": 20px Playfair Display, font-weight 600
- "CONSULTANT": 11px DM Sans, font-weight 600, letter-spacing 3px, uppercase

**Nav logo (top navigation bar):**
- Icon: 36px circle
- "The Restaurant": 15px Playfair Display, font-weight 600
- "CONSULTANT": 8.5px DM Sans, font-weight 600, letter-spacing 2.5px, uppercase
- This needs to fit comfortably in the existing nav bar height

**Icon only (favicon, app icon, mobile):**
- Just the chef hat inside the circle
- Must be legible at 16px, 32px, and 48px
- At 16px, increase stroke width so it remains visible

### Color Specifications

**On dark backgrounds (#0d1117):**
- Circle border: #22d3ee (cyan)
- Chef hat stroke: #22d3ee
- "The Restaurant": #ffffff
- "CONSULTANT": #22d3ee

**On light backgrounds (#f5f5f0 or white):**
- Circle border: #0e7490 (darker teal)
- Chef hat stroke: #0e7490
- "The Restaurant": #0d1117
- "CONSULTANT": #0e7490

## Typography Rules (for the whole app, not just the logo)

While updating the logo, also consolidate fonts across the app:

**Keep only these Google Fonts:**
- **Playfair Display** — for headings, the logo "The Restaurant" text, and page titles
- **DM Sans** — for body text, UI elements, buttons, form labels, and the logo "CONSULTANT" text

**Remove all other fonts from index.html.** The current font import loads ~30 families. Strip it down to just these two. This will dramatically improve page load speed.

Updated font import:
```html
<link href="https://fonts.googleapis.com/css2?family=DM+Sans:opsz,wght@9..40,300;9..40,400;9..40,500;9..40,600;9..40,700&family=Playfair+Display:wght@400;500;600;700&display=swap" rel="stylesheet">
```

Update CSS variables or Tailwind config to use:
- `--font-heading: 'Playfair Display', serif`
- `--font-body: 'DM Sans', sans-serif`

Apply consistently:
- Page titles and section headings → Playfair Display
- Everything else (body, buttons, labels, nav links, descriptions) → DM Sans

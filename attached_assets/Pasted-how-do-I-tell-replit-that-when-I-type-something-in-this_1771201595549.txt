You don’t “tell Replit” by a setting — you wire your textbox to an API route in your Replit app, and that route calls an LLM (OpenAI, etc.) server-side and returns a suggested caption. Replit is just hosting your code.

Here’s the clean, standard setup (works great for a “Facebook marketing manager” style caption generator).

1) Add an “AI Suggest” button next to that box (frontend)

When clicked (or on debounce while typing), call your backend:

// client/src/lib/generateCaption.js
export async function generateCaption({ prompt, platform = "facebook", tone = "smart-fun", business = "Mouton’s Bistro & Bar" }) {
  const res = await fetch("/api/social/generate", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ prompt, platform, tone, business }),
  });

  if (!res.ok) throw new Error(await res.text());
  return res.json(); // { caption, hashtags, altText?, ideas? }
}

Then update your UI state with the returned caption and show it in the preview panel.

2) Create a backend endpoint in your Replit server (Express example)

Important: Keep your API key on the server, not in the browser.

// server/routes/social.js
import express from "express";
import OpenAI from "openai";

const router = express.Router();
const client = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

router.post("/generate", async (req, res) => {
  try {
    const { prompt, platform, tone, business } = req.body;

    const instructions = `
You are a senior social media marketing manager for a restaurant.
Write 1 ${platform} post caption.

Rules:
- Sound like a real human, confident, not salesy.
- Hook in first line. 1–3 short paragraphs max.
- Include a clear call-to-action (reservation, visit, comment, etc.).
- Avoid making up facts (hours, prices, events). If needed, speak generally.
- Keep it "${tone}".
Return JSON with keys: caption, hashtags (array).
`;

    const response = await client.responses.create({
      model: "gpt-5",
      reasoning: { effort: "low" },
      instructions,
      input: `Business: ${business}\nUser idea: ${prompt}`,
    });

    // If you asked for JSON, you can parse. For simplicity, here we just return text:
    // Better: enforce JSON with a strict prompt + JSON.parse try/catch.
    const text = response.output_text;

    // Minimal “good enough” fallback:
    res.json({ caption: text });
  } catch (err) {
    console.error(err);
    res.status(500).send("Failed to generate caption.");
  }
});

export default router;

And mount it:

// server/index.js
import socialRoutes from "./routes/social.js";
app.use("/api/social", socialRoutes);

This is the current recommended “Responses API” pattern.

In Replit:

4) Make it feel like “typing generates suggestions”

Two good UX options:

A) Button-driven (simplest, safest):

User types idea → clicks “AI Assistant” → caption replaces/appears in preview.

B) Auto-generate with debounce (more magical):

On input change, wait ~600–900ms, then call /api/social/generate.

Show “Generating…” spinner.

Cache last prompt so you don’t spam calls.

5) Make the outputs reliably “FB manager quality”

Do these 3 upgrades next (they matter more than model choice):

Add safety/moderation guardrails if users can type anything (OpenAI has safety best practices + Moderation API guidance).

Backend (Express + TypeScript)
1) Install OpenAI SDK
npm i openai
2) Add a route in server/routes.ts

This keeps keys server-side and returns a structured result.

// server/routes.ts
import type { Express, Request, Response } from "express";
import OpenAI from "openai";

const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY });

type SocialPostRequest = {
  prompt: string;
  platform?: "facebook" | "instagram" | "google";
  tone?: "smart" | "smart-fun" | "upbeat" | "luxury" | "casual";
  businessName?: string;
  location?: string;
  cta?: string; // optional: "Reserve online" etc
};

export function registerRoutes(app: Express) {
  // ...your existing routes

  app.post("/api/ai/social-post", async (req: Request, res: Response) => {
    try {
      const body = req.body as SocialPostRequest;

      const prompt = (body.prompt ?? "").trim();
      if (!prompt) return res.status(400).json({ error: "Missing prompt" });

      const platform = body.platform ?? "facebook";
      const tone = body.tone ?? "smart-fun";
      const businessName = body.businessName ?? "Your Restaurant";
      const location = body.location ?? "";
      const cta = body.cta ?? "Come see us";

      const instructions = `
You are a senior social media marketing manager for a restaurant.

Write 1 ${platform} caption based on the user's idea.

Rules:
- Hook in the first line.
- Sound human, confident, not salesy.
- 1–3 short paragraphs max.
- Do NOT invent specifics (hours, prices, events). If the user didn't provide it, keep it general.
- Include a clear call-to-action (CTA).
- No excessive emojis (0–2 max).
- Tone: ${tone}

Return STRICT JSON only with:
{
  "caption": string,
  "hashtags": string[],
  "altText": string,
  "variations": string[]  // 2 short alternate captions
}
`;

      const input = `Business: ${businessName}
Location: ${location}
CTA preference: ${cta}
User idea: ${prompt}`;

      const response = await openai.responses.create({
        model: "gpt-5",
        reasoning: { effort: "low" },
        instructions,
        input,
      });

      const text = response.output_text?.trim() ?? "";

      // Try to parse JSON. If the model returns extra text, this will fail;
      // we do a cheap extraction fallback.
      let data: any;
      try {
        data = JSON.parse(text);
      } catch {
        const firstBrace = text.indexOf("{");
        const lastBrace = text.lastIndexOf("}");
        if (firstBrace !== -1 && lastBrace !== -1) {
          data = JSON.parse(text.slice(firstBrace, lastBrace + 1));
        } else {
          return res.json({
            caption: text,
            hashtags: [],
            altText: "",
            variations: [],
          });
        }
      }

      return res.json(data);
    } catch (err) {
      console.error("AI social-post error:", err);
      return res.status(500).json({ error: "Failed to generate post" });
    }
  });
}

Frontend (React + TS + Vite + React Query)
1) Create an API function
// client/src/api/ai.ts
export type SocialPostResponse = {
  caption: string;
  hashtags: string[];
  altText: string;
  variations: string[];
};

export async function generateSocialPost(input: {
  prompt: string;
  platform?: "facebook" | "instagram" | "google";
  tone?: "smart" | "smart-fun" | "upbeat" | "luxury" | "casual";
  businessName?: string;
  location?: string;
  cta?: string;
}): Promise<SocialPostResponse> {
  const res = await fetch("/api/ai/social-post", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(input),
  });

  if (!res.ok) {
    const msg = await res.text();
    throw new Error(msg || "Failed to generate");
  }
  return res.json();
}
2) Wire it to your textbox using useMutation

Button-driven (recommended first):

// client/src/components/SocialComposer.tsx
import { useState } from "react";
import { useMutation } from "@tanstack/react-query";
import { generateSocialPost } from "@/api/ai";
import { Button } from "@/components/ui/button";
import { Textarea } from "@/components/ui/textarea";

export function SocialComposer() {
  const [text, setText] = useState("");
  const [suggested, setSuggested] = useState("");

  const mutation = useMutation({
    mutationFn: generateSocialPost,
    onSuccess: (data) => {
      setSuggested(data.caption);
      // Optionally also put it into the main editor:
      // setText(data.caption);
    },
  });

  return (
    <div className="space-y-3">
      <Textarea
        value={text}
        onChange={(e) => setText(e.target.value)}
        placeholder="Type an idea…"
        className="min-h-[160px]"
      />

      <div className="flex gap-2">
        <Button
          onClick={() =>
            mutation.mutate({
              prompt: text,
              platform: "facebook",
              tone: "smart-fun",
              businessName: "Mouton’s Bistro & Bar",
              location: "Cedar Park, TX",
              cta: "Book a table or drop in",
            })
          }
          disabled={mutation.isPending || text.trim().length < 5}
        >
          {mutation.isPending ? "Generating…" : "AI Assistant"}
        </Button>

        <Button
          variant="secondary"
          onClick={() => navigator.clipboard.writeText(suggested)}
          disabled={!suggested}
        >
          Copy Caption
        </Button>
      </div>

      {/* Preview */}
      <div className="rounded-lg border p-3 text-sm">
        <div className="font-medium mb-2">Post Preview</div>
        <div className="whitespace-pre-wrap">{suggested || "No suggestion yet."}</div>
      </div>
    </div>
  );
}
If you want “as I type, generate suggestions”

Do it with a debounce so you don’t spam requests:

import { useEffect } from "react";

useEffect(() => {
  const clean = text.trim();
  if (clean.length < 12) return;

  const t = setTimeout(() => {
    mutation.mutate({
      prompt: clean,
      platform: "facebook",
      tone: "smart-fun",
      businessName: "Mouton’s Bistro & Bar",
      location: "Cedar Park, TX",
      cta: "Come see us",
    });
  }, 800);

  return () => clearTimeout(t);
  // eslint-disable-next-line react-hooks/exhaustive-deps
}, [text]);

Common gotchas (so you don’t waste time)

Do not call the LLM from the browser (key leak).

Ensure Express has app.use(express.json()) set before routes.

If Replit Auth protects routes, either:

keep this endpoint behind auth (fine), or

explicitly allow unauthenticated if you want public usage (not recommended).

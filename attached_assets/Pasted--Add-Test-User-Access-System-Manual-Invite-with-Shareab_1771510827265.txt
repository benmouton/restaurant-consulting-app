# Add Test User Access System — Manual Invite with Shareable Links

I need the ability to manually add test users and send them a direct link to access the app with a temporary test account. This should be simple and admin-only.

---

## What I Need

1. An admin-only page or section where I can create test access links
2. Each link gives the recipient full access to the app without needing to pay
3. Links should be shareable via text/email (just a URL I can copy and paste)
4. I should be able to see who I've invited, whether they've used their link, and revoke access

---

## 1. Admin Panel — Test User Management

### Add a new section to the Admin Dashboard (`/admin`)

**"Test Access Manager"** section with:

#### Create Test Access Form:
- **Name** (text field) — who is this for (e.g., "Rick")
- **Email** (text field, optional) — for tracking purposes
- **Access Duration** (dropdown): 7 days, 14 days, 30 days, Unlimited
- **Access Level** (dropdown): Full Access (all domains), Read Only (can view but not generate), Specific Domains (checkboxes)
- **"Generate Test Link"** button

#### When the button is clicked:
1. Create a unique token (e.g., UUID or random string)
2. Store in the database: token, name, email, access level, duration, created date, expiry date, status (active/used/expired/revoked)
3. Generate a URL like: `https://restaurantai.consulting/test-access/{token}`
4. Show the URL in a copyable field with a **"Copy Link"** button
5. Optionally show a **"Send via Email"** button if you have email configured

## 2. Test Access Route

### Create a new route: `/test-access/:token`

**File:** Add to `App.tsx` routing

When someone visits this URL:

1. Look up the token in the database
2. **If valid and not expired:**
   - Create a temporary user account (or session) with the specified access level
   - Set a flag on the account: `isTestUser: true`
   - Set the expiry date from the token config
   - Skip the subscription gate for test users — they should NOT see the subscription page
   - Redirect to the dashboard
   - Show a subtle banner at the top: "Test Access — Expires [date]. Enjoying this? Subscribe for $10/month →"
3. **If expired:**
   - Show a clean page: "This test link has expired. Want full access? Get started for $10/month →" with a subscribe button
4. **If invalid or revoked:**
   - Show: "This link is no longer valid." with a link to the landing page

## 6. Database Schema

### Add a `test_access_tokens` table (or collection):

```
- id (primary key)
- token (unique string, indexed)
- name (string)
- email (string, nullable)
- access_level (enum: full, read_only, custom)
- allowed_domains (array of domain slugs, nullable — only used if access_level is custom)
- duration_days (integer)
- created_at (datetime)
- expires_at (datetime)
- status (enum: active, used, expired, revoked)
- used_at (datetime, nullable — when the link was first clicked)
- user_id (foreign key, nullable — links to the user account created when they used the link)
- created_by (foreign key — the admin who created it)
```

---

## 7. Security Considerations

- Tokens should be long enough to prevent guessing (at least 32 characters)
- Rate limit the `/test-access/:token` endpoint to prevent brute force
- Test users should NOT be able to:
  - Access the admin panel
  - Create other test links
  - Modify subscription settings
  - See or interact with other users' data
- When a test user's access expires, their session should be invalidated on next page load
- All test user data (any documents they uploaded, SOPs they created, etc.) should be cleaned up after expiry OR preserved if they convert to paid

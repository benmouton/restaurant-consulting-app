# Build a Standalone Media Kit Page — NOT Connected to the Main App

Create a completely separate, standalone page at `/media-kit` that is NOT linked from the main app navigation, dashboard, or any other page. It should not appear in the nav bar, footer, or any menu. It is only accessible by directly visiting the URL. This page is a professional media kit / about page designed to be shared as a link on Facebook and social media.

**This is a public page — no authentication required.**

---

## Page Design

### Style
- Use the same dark theme as the rest of the app (#0d1117 background, white text, cyan #22d3ee accents)
- But this page should feel more editorial and personal — like a magazine profile, not a SaaS dashboard
- Use Playfair Display for headings, DM Sans for body text
- Full-width layout, generous whitespace, large typography
- Mobile-responsive — this will primarily be viewed from Facebook on phones

## Technical Requirements

1. Create this as a new route in the app: `/media-kit`
2. **Do NOT add it to the navigation bar, footer links, or any other navigation in the app**
3. It should be a public page — no authentication required (similar to `/privacy` and `/terms`)
4. Add the route to `App.tsx`:
```tsx
<Route path="/media-kit" component={MediaKitPage} />
```
5. Create the page component at `src/pages/media-kit.tsx`
6. The page should have its own layout — no sidebar, no app nav bar. It's a standalone landing page.
7. Include Open Graph meta tags so it previews well when shared on Facebook:
   - og:title = "Ben Mouton — Restaurant Owner, Operator, Consultant"
   - og:description = "14+ years in the restaurant industry. Builder of The Restaurant Consultant."
   - og:image = (use the app's favicon or logo for now)
   - og:url = the media kit URL
8. Make it print-friendly — operators may want to print or PDF this page
